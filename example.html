1. Page Structure

Every page on this site uses the same basic HTML skeleton. Semantic elements like <header>, <nav>, and <main> divide the page into meaningful sections that are easy to read and style.

<body>

    <header>
        <h1>Welcome to Egypt</h1>
    </header>

    <nav>
        ... navigation links ...
    </nav>

    <main>
        ... page content ...
    </main>

</body>

2. Linking a Stylesheet

All five pages on this site share a single CSS file called styles.css. The <link> tag inside <head> tells the browser where to find it. This means any change to styles.css instantly applies to every page.

<head>
    <link rel="stylesheet" href="styles.css">
</head>

3. Background Images

The site has two background images: one on the <body> that fills the entire page, and one on the <header> that acts as the hero image. Both use background-size: cover to make the image fill the space without leaving any gaps, no matter the screen size.

body {
    background-image: url(Images/backgnd1.webp);
    background-size: cover;
    background-repeat: no-repeat;
}

header {
    background-image: url(Images/hero.webp);
    background-size: cover;
    background-repeat: no-repeat;
}

4. The Dark Overlay Trick

White text on a light photo is impossible to read. To fix this, a semi-transparent black layer is placed over the background image using linear-gradient with rgba color. The fourth number in rgba controls transparency — 0 is invisible, 1 is fully opaque.

header {
    background-image: linear-gradient(rgba(0,0,0,.5), rgba(0,0,0,.5)),
                      url(Images/hero.webp);
}

5. Flexbox

Flexbox is used in two places: the navigation bar and the image gallery. By default, HTML elements stack vertically. Adding display: flex to a container arranges its children side by side in a row. justify-content: center then centers them horizontally.

nav > ul {
    display: flex;
    justify-content: center;
    align-items: center;
}

6. The Image Gallery: Divs as Boxes

A <div> is a generic container — an invisible box you can put anything inside. The image gallery on the homepage uses two layers of divs nested inside each other. The outer div (.pics) holds all three images in a row using flexbox. Inside it are three inner divs (.container), and each one holds a single <img>.

<div class="pics">

    <div class="container">
        <img src="Images/cairo.webp">
    </div>

    <div class="container">
        <img src="Images/sphinx.webp">
    </div>

    <div class="container">
        <img src="Images/statues.webp">
    </div>

</div>

The reason for the inner .container div is to control the size. We give the container a fixed width and height, and then tell the image to fill it completely. This way CSS is sizing the box, and the image just fills whatever space the box takes up.

.pics {
    display: flex;        /* lines the containers up side by side */
    height: 40vh;
}

.container {
    width: 25%;           /* the box takes up 25% of the row */
    height: 75%;
}

.container > img {
    width: 100%;          /* image fills the box */
    height: 100%;
    object-fit: cover;
}

7. object-fit: cover

The three photos on the homepage all have different original dimensions, but they need to fill their containers perfectly. object-fit: cover tells the browser to crop the image to fit the space while keeping its original proportions — no stretching.

.container > img {
    width: 100%;
    height: 100%;
    object-fit: cover;
}

8. The Active Page Trick

Each page highlights its own link in the navigation bar, so visitors always know where they are. This is done by adding id="active" to the correct <a> tag on each page, then styling that id differently in CSS.

<!-- On the Pyramids page, only that link gets the id: -->
<li><a href="index.html">Home</a></li>
<li><a id="active" href="page1.html">The Pyramids</a></li>
<li><a href="page2.html">The Nile River</a></li>

/* In styles.css: */
#active {
    color: black;
    text-shadow: none;
}
